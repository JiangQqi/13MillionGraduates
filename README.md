# 《一千三百万毕业生》 / 13 Million Graduates

> ⚠️ 本 README 由 DeepSeek 辅助生成。

一款受《人力资源部》（Human Resource Machine）启发的编程解谜游戏，使用 Unity 从零复现。

## 玩法

你扮演一名流水线工人，站在格子地毯前，面对传送带上不断涌来的数据方块。

你的任务不是亲自搬运——而是**写汇编程序**，让角色按你的指令自动完成分拣。

画面左侧拖拽指令块拼成程序，点击运行，小人就会逐条执行：走到输入栏取方块、复制到指定位置、做算术运算、按条件跳转——直到所有数据正确输出。

## 核心技术

**整个游戏的灵魂，是一套完全自主实现的汇编语言。**

### 自定义指令集

七条指令：

| 指令 | 作用 |
|------|------|
| `INBOX` | 从输入栏取一个数据方块 |
| `OUTBOX` | 将手中方块放入输出栏 |
| `COPYTO [n]` | 将手中方块复制到 n 号位（支持间接寻址） |
| `COPYFROM [n]` | 从 n 号位读取方块到手中 |
| `ARITHMETIC op [n]` | 12 种运算：ADD / SUB / MUL / DIV / INC / DEC / EQL / NEQ / GTR / LES / GEQ / LEQ |
| `JUMP type label` | 条件跳转：ANY（无条件）/ ZRO（为零时）/ NEG（为负时） |
| `label:` | 跳转锚点 |

可变存储、条件跳转、算术运算三者齐备，具备图灵完备的全部能力。

### 编译执行管线

- **CodeExecutor**：线上虚拟机，驱动角色动画，按流程执行指令
- **CodeSerializer**：双向序列化引擎，代码块 ↔ 文本互相转换，支持剪贴板导入导出
- **CodeSimulator**：离线仿真器，不依赖任何 Unity 对象，纯逻辑对多组随机测试数据盲跑验证

### 状态机动画系统

基于 `StateMachine` + `StateQueue` 架构驱动角色行为：

每个指令对应一个 State，动画完成后自动推进到下一条指令，状态与动画完全解耦。

### 动态类型系统

所有数据方块底层统一存储为字符串。运行时自动推断类型——整数、布尔值、字符（百家姓 + 英文字母）——并按需转换。不同类型之间可直接进行四则运算和比较：系统统一转为整数计算，再按规则类型输出。自定义 ASCII 映射表覆盖中英文字符。

### 双层通关验证

画面上的通关只是表象——后台 `CodeSimulator` 会用多组随机测试数据反复重跑你的程序，**只有对所有输入都输出正确结果，才算真正过关**。

### 逐级开放指令集

每一关可独立配置允许使用的指令及子类型（算术四则 / 自增减 / 比较分别开关，跳转 ANY / ZRO / NEG 分别开关，间接寻址独立开关），渐进式解锁，导入外部代码时也会校验当前关卡的指令权限。

1. 使用 Unity 6000.0.23f1 或更高版本打开 `Unity/13MillionGraduates`
2. 打开 `Scenes/DefaultLevel_N` 场景
3. 点击 Play

## 已知问题

以下功能因开发时间紧迫暂未完成，不影响正常游玩流程：

| 问题 | 说明 |
|------|------|
| **Step 调试未实现** | `CodeExecutor` 的 StepPrev / StepNext 均为空 TODO，目前只能全量运行，无法单步调试 |
| **Undo 未实现** | `CodeManager` 的撤销按钮已预留但逻辑为空 |
| **State 实例复用** | `StateMachine` 使用单例模式复用 State 实例，理想做法是每次 Execute 时 new 新实例（源码中已标注 TODO） |
| **Advanced 模式未完成** | `AreaManager` 的区域划分系统已搭建好框架，但 CopyTo / CopyFrom / Arithmetic 的具体操作逻辑在 Advanced 分支中均为空 `break;`，目前仅 Classic 模式可用 |
| **部分动画存在轻微视觉瑕疵** | 少量动画过渡有微小视觉 Bug，不影响游戏逻辑 |
